ADB指令刷入recovery、获取root权限、刷机

发表于 2014-09-02   |   分类于 Android   |  
  • 什么是ADB
    ADB: Android Debug Bridge 意为安卓调试桥。Adb是android sdk 里面的一个工具，通过它可以命令行的方式直接操作管理android模拟器或者真实的andriod设备。这个过程中pc为客户端，安卓设备为服务端。利用常用的ADB指令可以实现查看设备、传输文件、登录shell、安装卸载软件等一系列的操作。

在手机党刷机篇介绍了利用现成的工具和教程实现root及刷机的思路，但是远没有通过ADB操作的高效。可能当你正愁着怎么同时多按键进入fastboot(或者recovery)模式时，一个 adb reboot bootloader(recovery) 命令就已经解决了。

相关名词解释：

fastboot:顾名思义，快速启动。在此模式下进行线刷。（adb reboot bootloader进入fastboot模式）

recovery: 在此模式下进行卡刷。需要刷入cwm等第三方recovery实现第三方rom包的刷入。（adb reboot recovery 进入recovery模式）

bootloader: 这个相当于pc的bois，为设备的第一启动加载阶段，有时需要解锁。

不同机型，情况不一。以三星i9300为例，拥有挖煤模式（downloading模式），进入的命令为：

adb reboot download 或者adb reboot bootloader

可见，或许三星的挖煤模式就是fastboot模式。大家可以着重参照自己的机型来理解这几个概念。

  • 刷入recovery
    准备工作：安装手机设备驱动，打开设备USB调试功能，adb+fastboot工具包及使用方法，准备好第三方recovery包(recovery.img)。

进入fastboot模式：adb reboot bootloader

刷入recovery: fastboot flash recovery recovery.img (文件的路径按照实际情况)

重启机器后，验证是否刷入成功：adb reboot recovery

自此手机已经具备了卡刷的能力。
