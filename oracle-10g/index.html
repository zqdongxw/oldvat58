oracle 10g 客户端服务端配置全教程

发表于 2012-09-16 | 分类于 数据库 | |

详细的图文流水教程可以下载文章后面的文档，本文介绍下配置过程中需要注意的问题。

①在安装oracle 10g时，在先行条件检测时会出现“正在检查网络配置需求……“。检查完成后此次检查的总体结果为：未执行。

解决方案：

xp：控制面板->添加硬件，在向导中选择“是，已经连接了此硬件”→添加新硬件设备→安装我手动列表选择硬件(高级)→网络适配器→厂商：Microsoft 网卡：Microsoft Loopback Adapter，添加完成后，就可以配置下这个虚拟本地连接。

windows7：添加新硬件向导可能不太好找，直接运行命令：hdwwiz。打开向导后，接下来你就熟悉了。

我们的目的达到了，通过oracle的先决条件检查，避免一些未知的错误。

②手动创建数据库时，可以选择数据库模板，四个模板都是什么意思呢？

一般用途：创建的数据库可以适合作为事务处理应用，也可以作为数据仓库应用，两种应用兼而有之，这是数据库安装时的默认选项。

事务处理：侧重于日常业务应用；从物理上讲，数据块小，uid和pid都小，侧重读写速度，冗余度小；适合联机事务处理（OLTP）数据库应用，比如各银行的储蓄业务系统、电信局的电话计费系统就是典型的OLTP的例子。

定制数据库：自己定制所需要的模板。呵呵，以后再学习定制吧。

数据仓库：侧重于数据分析统计；从物理上讲，数据块大，uid和pid都大，侧重分析研究，冗余度大；适合数据仓库及联机分析处理（OLAP）应用，比如银行的管理层要制订业务发展计划时，决策人员就可以从数据仓库中提取有利于决策的数据。

似懂非懂的理解下先。

③sqlplus命令行中常见的错误举例：RA-12560: TNS: 协议适配器错误

可能的原因有：
1.监听服务没有启动。没有启动，还是服务列表里压根就没有这个服务？两个命令帮助你：
lsnrctl status 查看监听器状态
lsnrctl start 启动监听器

2.oracle的数据库实例没有启动。在服务列表里检查启动oracleserviceXXXX（XXXX为SID）

④安装完oracle后，net manager 中的配置是非常重要的一步。

 

服务端中，配置 【服务命名】和【监听程序】，在orale主目录\NETWORK\ADMIN里会分别产生tnsnames.ora ，Listener.ora 两个文件。

客户端中，只需要配置【服务命名】。这是客户端连接服务端的关键。

⑤创建表空间和用户。

创建数据库完毕后登陆测试，以及创建表空间和用户：

C:\Documents and Settings\Administrator>sqlplus /nolog
 SQL*Plus: Release 10.2.0.4.0 - Production on 星期四 2月 23 07:48:08 2012
 Copyright (c) 1982, 2007, Oracle.  All Rights Reserved.
 SQL> conn / as sysdba
 已连接。

创建表空间

SQL> create tablespace test datafile 'E:\ORACLE\PRODUCT\10.2.0\ORADATA\TEST\test.dbf' size 30000M;
表空间已创建。

创建临时表空间

SQL> create temporary tablespace test_temp tempfile 'E:\ORACLE\PRODUCT\10.2.0\ORADATA\TEST\test_temp_01.dbf' size 5000M;
表空间已创建。

创建用户：

SQL> create user vat58 default tablespace test temporary tablespace test_temp
identified by 123;
用户已创建。

给用户赋权：

SQL> grant dba to vat58;
授权成功。

使用创建用户登陆：

SQL> conn vat58/123
已连接。
