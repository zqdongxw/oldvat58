Android学习笔记系列07–Handler

发表于 2012-06-06   |   分类于 Android   |     |  

Android中Handler是一个较为复杂的概念。Android developers 的解释为：

A Handler allows you to send and process Message and Runnable objects associated with a thread's MessageQueue.

Handler主要接受子线程发送的数据, 并用此数据配合主线程更新UI。它的本意是将一些复杂的或者说耗时的操作放在子进程中进行，实现异步的操作。

使用实例：

public class HandlerDemoActivity extends Activity {
    private Button but1 = null;
    private Button but2 = null;
    private ProgressBar pb = null;
    Handler handler = new Handler();
    /* Called when the activity is first created. /
    @Override
    public void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);
        setContentView(R.layout.main);
        but1 = (Button)findViewById(R.id.but1);
        but2 = (Button)findViewById(R.id.but2);
        but1.setText("START");
        but2.setText("END");
        pb = (ProgressBar)findViewById(R.id.pb);
        pb.setProgress(0);
        but1.setOnClickListener(new OnClickListener(){

        public void onClick(View v) {
            // TODO Auto-generated method stub
            handler.post(add);
        }

    });
    but2.setOnClickListener(new OnClickListener(){

        public void onClick(View v) {
            // TODO Auto-generated method stub
            handler.removeCallbacks(add);
        }

    });

}
int pro = 0;
Runnable add = new Runnable(){

    public void run() {
        // TODO Auto-generated method stub
        pro = pb.getProgress()+1;
        pb.setProgress(pro);
        setTitle(String.valueOf(pro));
        if (pro&lt;100){
        handler.postDelayed(add, 1000);
        }
    }

};

}
例子中结合了之前介绍过的progressbar和button的应用。演示效果如下：

Handler关联的是一个单独的线程和消息队列，但默认关联的是主线程。所以在上例与我们的初衷是不同的，它并没有Start一个新的进程，handler所处理的操作依然是在主进程中执行的。我们可以使用HandlerThread来启动一个新的线程。

HandlerThread thread = new HandlerThread("newthread");
thread.start();

还有几个比较重要的概念：looper，message，messageQueue 注意学习区分。
