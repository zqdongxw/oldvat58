WordPress之创建子主题

发表于 2012-03-25 | 分类于 wordpress |

Wordpress是一款强大的个人博客系统，如果你已经在写一个wp的博客，那么恭喜你：你己经站在了巨人的肩膀上。我们有理由把它完善地更好，更符合个人的要求和特色，所以有时候我们需要自己DIY。可你 有没有想过，你的“定制”和修改会随着下一次的升级付之东流，我确实担心过。

创建wordpress的子主题，会让这个问题迎刃而解。

WordPress子主题也是一个主题，它继承了另一个主题——父主题——的功能，并允许你对父主题的功能进行修改，或者添加新功能。

style.css是一个子主题唯一必须的文件。它的头部提供的信息让WordPress辨认出子主题，并且重写父主题中的style.css文件。

这样一个子主题就建好了。对一个父主题的样式和布局进行修改和扩展，而不需要对父主题的文件作任何修改。通过这样的方式，当父主题被更新的时候，所做的修改就可以保存下来。所以强烈建议以这种方式对主题进行修改。

你的子主题文件夹里面可以少至只包含一个style.css文件，也可以包含多至一个完整WordPress主题所拥有的文件：

1.style.css (必需)


/
Theme Name: Vinny
Theme URI: http://www.vat58.com/
Description: the test one by vinny_dong
Author: dong
Version: 1.0
Template: twentyten
/


以上头部信息解释为：


Theme Name. (必需) 子主题的名称 vinny。 Theme URI. (可选) 子主题的主页 www.vat58.com。
Description. (可选) 子主题的描述。比如：the test one by vinny_dong Author. (optional) 作者的名字 dong。
Template. (必需) 父主题的目录名，区别大小写。Version. (可选) 子主题的版本。比如：0.1，1.0，等。
可以用import命令将模板的css导入，然后修改样式规则，覆盖同名的样式规则

如：@import url(“../twentyten/style.css”); （此语句必须在所有规则之上才能起作用）

2.functions.php (可选)

不像style.css，子主题中的functions.php不会覆盖父主题中对应功能，而是将新的功能加入到父主题的functions.php中。（其实它会在父主题文件加载之前先载入。）
<?php

Function newFunction() {}

?>

可以以上述方式给主题添加功能。你也可以对父主题中的一个PHP函数重新声明替换。

3.Template files (可选)

在子主题中的表现和style.css一样，它们会覆盖父主题中的相同文件。子主题可以覆盖任何父主题模板中的文件，只需要创建同名文件就行。

相信接下来你肯定有很多工作要做了，定制一个*子主题，将以前改动的地方体现在你的子主题文件里，这样才可以后枕无忧。
