有包参与时，类路径（classpath）的问题

发表于 2012-04-12 | 分类于 Java |

如果用package语句指明一个包，则包的层次结构必须与文件目录的层次相匹配。

比如：

F:\test\Test.java
package test; // 包的层次结构与文件目录的层次相匹配
public class Test {
Test () {
System.out.println(“test constructor”);
}
public static void main(String[] args) {
Test x = new Test ();
}
}

下面编译执行java（日文系统中”¥”代表”\”）

F:\test>javac Test.java


成功编译，得到Test.class

如果直接 F:\test>java Test 会报错：找不到Test 类

需要注意的两点：

  1. 运行包中的类时，需要指明包名，也就是要指明test.Test
  2. 当前的类路径为classpath=.; 也就是当前目录。本例中由于包的参与，执行时寻找的路径为F:\test\test，而这个目录是不存在的。
    于是解决办法：

1． F:\test>java -classpath F:\ test.Test
用-classpath选项指定类路径为当前目录的上一层目录F:\

2． 到上一层目录F:\执行命令


得到正确结果。

或者在classpath中包含当前目录的上一层记录。

 

 

WordPress RSS 初步优化

发表于 2012-04-12 | 分类于 wordpress |

认识RSS

RSS全称为Really Simple Syndication，称之为聚合内容。网站提供RSS输出有利于让用户获取网站内容的最新更新，这样读者就可以根据自己的需要订阅信息，在最短的时间内获取最有价值的信息。在博客时代，RSS已经成为不可缺少的工具。

wordpress RSS 优化

在《WordPress完善笔记》一文中提到，chrome下的wordpress原有的RSS订阅功能异常。不仅如此，wordpress自带的Rss功能较简陋，有诸多不便。于是用feedsky来管理我的feed（feedburner被墙，国内使用不太方便）。Feedsky目前提供的服务包括Feed统计、优化和发布三部份，Feed的优化和传播管理得到了加强。

去feedsky管理你的wordpress rss feed

在feedsky发布中可以定制你喜欢的形式。我的feed发布：

重定向WordPress Feeds 到申请的Feedsky地址，这样就不用把_bloginfo(‘rss2url’) 换成自己的feed地址，省去了系统升级或主题更换带来的手动修改的麻烦。

具体操作：新建或修改.htaccess文件，添加如下内容（粉色部分替换成自己的feed地址）

temp redirect wordpress content feeds to feedsky

<IfModule mod_rewrite.c>
RewriteEngine on
RewriteCond %{HTTP_USER_AGENT} !FeedBurner [NC]
RewriteCond %{HTTP_USER_AGENT} !FeedValidator [NC]
RewriteRule ^feed/?([_0-9a-z-]+)?/?$ http://feed.feedsky.com/vat58 [R=302,NC,L]
</IfModule>

保存.htaccess上传到网站根目录。

最后，删除默认的小工具“功能”中”文章RSS”，“RSS评论”和“wordpress.org”等多余选项：在wp-include内的default-widgets.php里删除以下内容：

  1. <li><a href=”<?php bloginfo(‘rss2_url’); ?>” title=”<?php echo esc_attr(__(‘Syndicate this site using RSS 2.0’)); ?>”><?php _e(‘Entries <abbr title=”Really Simple Syndication”>RSS</abbr>’); ?></a></li>
  2. <li><a href=”<?php bloginfo(‘comments_rss2_url’); ?>” title=”<?php echo esc_attr(__(‘The latest comments to all posts in RSS’)); ?>”><?php _e(‘Comments <abbr title=”Really Simple Syndication”>RSS</abbr>’); ?></a></li>
  3. <li><a href=”http://wordpress.org/“ title=”<?php echo esc_attr(__(‘Powered by WordPress, state-of-the-art semantic personal publishing platform.’)); ?>”>WordPress.org</a></li>

JDK环境变量设置与“Could not find main class”错误

发表于 2012-04-11 | 分类于 Java |

JDK环境变量如何设置

JAVA_HOME = X:\jdk1.6 (注：新建系统变量JAVA_HOME，其值”X:\jdk1.6”为jdk文件夹的绝对路径)

Path = …; %JAVA_HOME%\bin (注：在path变量中增加路径”%JAVA_HOME%\bin”，用”;”与其他路径隔开)

ClassPath = .;%JAVA_HOME%\lib\tools.jar;%JAVA_HOME%\lib\dt.jar (注：新建变量ClassPath)

另：环境变量不用区分大小写，能统一使用即可。

为什么这样设置

1. JAVA_HOME指向jdk的安装目录，便于eclipse,netbean,tomcat等工具寻找jdk;同时可以简化Path和ClassPath路径，一旦jdk路径改变，只需更改JAVA_HOME。

2. 如果不在path中添加jdk的bin目录，那么在cmd中javac，java等指令无法使用，显示为不是内部命令，也不是外部命令。这一步就是为了让java在cmd中”有处可寻“。

3. . 代表命令提示符的当前目录（注意）。tools.jar，dt.jar 为sun提供的类包，一般会加上。如果有了自己的类包，也可以将路径加上。classpath就是让java开发中要引用的类”有处可寻“。

附：

Classpath中要加入的jar包

rt.jar: java基础类库

dt.jar: 运行环境的类库

tools.jar: 工具类库

如果要搞清楚该不该加，就必须清楚每个jar包中包含的功能（类）。如果你写一个简单的helloworld，暂时不加肯定没问题。

为何会出现“Could not find main class”错误

首先确定，但你编译完Test.java后，没有出现”java Test.class“这样的低级错误（应该是”java Test“），否则会出现如题的错误。一般的情况是classpath的问题。在设置classpath的时候你应该注意到上文中红色的”.“了，表示将命令提示符的当前目录加入搜索路径。如果不是这样的话，java就找不到编译完成的类。另外，如果没有设置classpath,也可以利用javac ,java命令的相关选项指定搜索路径避免找不到类的错误。

如若能知其所以然，就不用”死记硬背“jdk的环境配置，被小而难缠的细节阻碍你前行的脚步。

初识JavaScript

发表于 2012-04-06 | 分类于 JavaScript |

将**javascript**放在何处？

使用<script>标签将javascript插入html页面：

Head部：当脚本被调用时，或者当事件被触发时，脚本就会被执行。把脚本放置到 head 部分，就可以确保在需要使用脚本之前，它已经被载入了。

Body部：在页面载入时脚本就会被执行。

Tips（注意区分head部跟body部的不同点）

外部js文件：如果在多个页面中需要相同的javascript，简单的办法就是引用外部的js文件。给<script>标签指定src属性就可，同时这种方法叫容易管理和维护。

应对老的浏览器

那些不支持 JavaScript 的浏览器会把脚本作为页面的内容来显示。我们可以使用HTML 注释标签来避免这个问题。

Javascript函数：由function关键字定义。

Javascript事件：当用户进入或离开页面时就会触发 onload 和 onUnload 事件，可以用来处理cookie，同时onload事件可以用来检测访问者的浏览器类型和版本，然后载入特定版本的页面;onSubmit 用于在提交表单之前验证所有的表单域;onMouseOver 和 onMouseOut 表示鼠标相关的事件。

怀着各种心情品读了《天道》这部电视剧，每天一到两集的进度慢慢将剧情展开，将丁元英的睿智达观展开，将某些悲剧展开。我在剧情的湍流中浮沉着，感受着周围依旧的环境，迫切想体会出什么东西。

芮小丹，当生则生，当死则死，来去自如，这个被丁元英认为不简单的丫头，呈现给众人的永远是一种气质美，一种沐浴过智慧芬芳油然而生的女性魅力。她想追求依附的是像丁元英那样的高度。她过于圣洁，因而注定是天国的女儿，是一种可望而不可及的美好。上帝送她来到世间，让周围的人惊羡于她的存在，又恰到好处地将她带走，留给人一种空落落的感觉。丁元英尚是如此，更何况我辈？很难说清楚小丹跟元英谁更精明，谁更有大智慧.

林雨峰死了，刘冰跳楼了，这两个可以勉强作为我辈代表人物的角色下场都不是很好。确切的说，刘冰才是真正作为我辈代表的角色之一吧。他跟我们一样，拥有人类启蒙前的愚蠢，贪婪，渴望得到地位、尊重、社会的认可。然而经过检验，他并不是一个合格品，而是一步步的把自己推向坟墓。我们每一个人都可以或多或少的从他身上找到自己的影子。从他身上，我们又能得到什么教训呢？表面上看，刘冰的死跟丁元英似乎有关联，可丁元英并不是凶手，在这一点上，丁元英只不过是规律的化身而已！如果我们可以达到林雨峰那样的事业高度也该知足了吧。可他又是怎样被憋死的呢？中国音响界的领军人物难道就这样不堪一击？他的对手仅仅是丁元英吗？两败俱伤，只要我比你多一口气，我就是赢家。或许刘跟林都应该好好学习下丁元英的那句话：神即道，道法自然，如来。而丁元英恰恰是掌握了道，掌握了规律醉心于音乐茶道之间的明白人。林刘不该跟规律作对，不该跟内心的道作对。世事纷扰，人心各异，作为人有很多处事原则，可最最根本的东西到底在哪里。是天道？人道？ “神即道，道法自然，如来”这句我看来神秘的话，在似懂非懂里我知道里面蕴含着无限的能量！

发生了这么多的事，但从丁元英的角度来看仅仅是一个假期而已，从哪来回哪去，仅此而已？

愚人节Google“情趣外衣”——水下搜索

发表于 2012-04-01 | 分类于 google |

 

我有一个不小的遗憾，而且看形势这遗憾还在蔓延：Google离开中国大陆后，我才认识并爱上她。每一次技术的创新与领跑，每一次舒适的用户体验和人文关怀，每一次精致到细节的惊喜，都让我对翻墙乐此不疲。Google你真的是华丽了全球，温暖了人心。我不是追星族，崇洋派，我只是被她魅惑了。

2012年的愚人节，Google穿出了她的“情趣外衣”——水下搜索。用你的鼠标尽情点吧，晃吧。说不定，某一点就能惊喜到你。我大概点了半个多小时…

如上，Google真的有如一个巨大的海洋宝藏，蕴藏着无限的能量。拍Google的“G点”容易，好好向她学习才是真道理。

什么？Google要返回中国啦？

真的吗？

愚人节，我在骗自己。唉，不过真的很希望。Google一直走“sexy”路线，我要追随她！
