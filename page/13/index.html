在工作中创造精确

发表于 2012-04-27 | 分类于 V文摘 |

人物周刊：拿到剧本，认领了角色，通常你会做哪些准备？

王志文：我现在的生活，除了演戏，基本上两点一线：儿子，球场。我跟孩子玩，脑子里会琢磨；到球场打球，不用跟人说话，也在慢慢进入。首先是理解角色，接收他，剔掉个人偏好、习惯地接收，同时让那部分变得丰富、生动起来。心里要有——你得满满当当才能表达，否则只能是敷衍。但现在拍戏大多是每天得拍掉多少页，这个景拍完了，马上换下一个，都游击队了，这事儿就比较麻烦了。同行说你太使劲儿了，可我觉得还不够。我的方式还是原来的，准备得充分一点。

人物周刊：《南行记》演艾芜那会儿，跟老先生相处了一段时间，现在还会有这类功课吗？

王志文：依然会。但年轻时跟现在准备的方式会不太一样。二十四五岁我可能需要坐下来跟艾老面对面聊，但四十多岁这个方式不见得是最好的，毕竟对老人是一种打扰。你的阅历、经验、理解力、想象力，都可以成为“准备”时可调动的资源。

人物周刊：对那些可能在经验之外的呢？比方《黑冰》里的毒枭郭小鹏。

王志文：芸芸众生，其实不分正面反面，每个人都是一个糅杂的打包，每个人也都是复杂而说不清的；但人类又有一些东西是本质的、相通的，所谓人性。所以不管什么角色，我只要把他当一个人去演就好了。

我可能就应该是做演员的人。有时候我不需要去跟某一类人具体接触，就可以体会到他。借助想象力，还有你的心。为什么我们都说妈妈做的饭菜好吃？因为那里面有她的心。

现在回想起来，郭小鹏是一个特别专情的人，他对什么事情（他的职业、他的女人）都很专注——这跟我有共通的地方。那么我就抓住这个点，让人物的力量流出来。想要一个什么感觉呢，就是如果这么一个人，他如果不从事制毒贩毒，该有多美妙。这样，他跟观众的关系就建立起来了（ 注：自身实践证明，这种关系建立的相当准确。读郭小鹏）。
（信息来源：南方人物周刊 ）

他的说话风格跟剧中是一致的，很多观众跟我一样喜欢王志文剧中的演绎风格：深沉，朴实。我不追星，甚至有时候还会解释，我喜欢的是他演绎的人物。仔细想想，要想把某种角色诠释的立体、神似、有血有肉，与观众建立起直接的关系，何尝不需要相应的经验和理解力，何尝不需要人物本身的“功力”？甚至只有你的“功力”高于待演绎的角色，才能够信手拈来，游刃有余。用他自己的话说，“你得满满当当才能够表达，否则只能是敷衍”。所以，王志文是一个我喜欢的不简单的演员。王大叔在演员生涯中创造着精确，我们有不少可以学习的地方：专注，在点上发力才能够精彩！这种感觉跟王志文的戏一样有味道。

feedsky绑定二级域名后不能更新有解啦

发表于 2012-04-23 | 分类于 wordpress |

在《cpanel设置二级域名与WordPress Rss优化》中，兴冲冲地跟大家分享了rss的终极优化方案。我本人也在不断的学习体验中。接下来的这一星期，feedsky不能更新了，郁闷了一个“十一长假”的阵容。现在终于解决了，特来相告。

这段时间，feedsky中feed状态一直处于失败状态。最近的一条让人欣喜，正常。

feedsky现在的状态就是安安静静地提供一个最简陋的rss服务，其他一概不管。客服论坛里都是发贴者在自己的帖子里自言自语：怎么没人回答呢。可以上溯到2008年之前了。想要让它给点帮助是不可能了。于是一遍遍地联系主机空间商，各种措施都失败了，在此感谢老薛主机技术人员的帮助。

见到“上述情况”，替我问一句，还记得在capnel中设置二级域名时生成的”feed“文件根目录吗？我设置的二级域名为feed.vat58.com，于是在主机目录里多了一个feed文件夹。这就是原因所在。

删掉feed文件夹，RSS从此顺畅无阻！(追记：长时间以来，feed文件夹是删了生，生了删，斗争了很久。稳妥的办法是更改二级域名的文件根目录，具体是由feed改成其他如feeds就好，避免与wordpress feed源的冲突。)

本文算是对《cpanel设置二级域名与WordPress Rss优化》一文的补充，feed.vat58.com正式启动，你即将订阅的是”成长“！

wordpress中小试HTML5

发表于 2012-04-22 | 分类于 HTML5 ， wordpress |

尽管HTML5到目前为止仍然是下一代html的草案，它的相关教程已经是铺天盖地。
话说HTML5前景一片光明， 给个人站点披上HTML5的外衣，逐步学习体验新的特
性，既紧随了潮流，又不失为一个好的学习方法。还在等什么？本文讲一下wordpress
中小试HTML5。

几点说明：
1. 本次示例的为wordpress 3.0 twentyten 主题，编辑的相关文件在“/wp-content/themes/twentyten/ ”目录。
2. 下文中提到的行号为修改代码的大体位置，请根据自己的实际灵活操作。

<article>标签

<article>标签定义外部的内容。外部内容可以是来自一个外部的新闻提供者的一篇新的文章，或者来自 blog 的文本，或者是来自论坛的文本。亦或是来自其他外部源内容。我们可以将此标签用于wordpress的文本中。

loop.php:
<div id=”post-<?php the_ID(); ?>” <?php post_class(); ?>> （61行左右）
=>(变更为)
<article id=”post-<?php the_ID(); ?>” <?php post_class(); ?>>
关闭标签处：
</div><!– #post-## –>
=>(变更为)
</article><!– #post-## –>

此文件的其他地方以及其他文件如，loop.php、loop-single.php、loop-page.php等等都有可能需要作类似的改动。

<header> ,<nav> ,<footer>

<header> 标签定义文档的页眉（介绍信息）。
<nav> 标签定义导航链接的部分。
<footer> 标签定义 section 或 document 的页脚。
<section> 标签定义文档中的节（section、区段）。比如章节、页眉、页脚或文档中的其他部分。

header.php:
<div id=”header”> （57行）
=>
<header id=”header”>

<div id=”masthead”> （58行）
=>
<section id=”masthead”>

<div id=”access” role=”navigation”> （81行）
=>
<nav id=”access” role=”navigation”>

footer.php:
<div id=”footer” role=”contentinfo”> （15行）
=>
<footer role=”contentinfo”>

附：关闭标签处不要忘记更改关闭标签。

虽然都是些细小的改动，外观上可能没有明显的效果，但是你已经在体验学习html5了，这将是网站彻底革新的开端。

php之函数

发表于 2012-04-18 | 分类于 PHP |

php中有超过700个内置函数，这或许是php最给力的地方。

如何创建自己的函数？
关键字：function
function functionname () {
/函数体/
}

php中没有函数重载的概念
不像Java，c 等高级语言，可以使用相同的函数名称，以参数的类型、个数、位置区分来实现函数重载。这在php中是不可以的。在php中调用函数时可以有任意的参数。
<?php
function demo($a) {
$args=func_get_args();
echo($a);
echo($args[1]);
}
demo(“vat58”,”每天一点PHP”);
?>
执行上述代码，页面上输出 vat58每天一点PHP
我们看到，尽管定义函数demo时只有一个形参，而在调用时却给它传递了两个参数。
这时没有发生错误，而是第一个实参传给了$a，其他参数可通过函数 func_get_args()
获得。func_get_args()返回值是一个数组，下标从零开始。于是$args[1]取到了第二个
参数“每天一点PHP”。

引用函数include()和require()
很多时候，我们要实现代码的重用，提高编写效率，就要在一个文件中引用其他文件
中的函数等等， 这时要用到include()或require()函数，相当于java中的import语句。
include(和require()唯一的区别在于对错误的处理方式上：
include() 函数会生成一个警告（但是脚本会继续执行），而 require() 函数会生成一个
致命错误（fatal error）（在错误发生后脚本会停止执行）。

如何绑定？

FeedSky有免费域名绑定服务,可以将子域名绑定到个人FeedSky的帐号。

依据图中操作步骤，结合Cpanel，我们首先在DNS中添加子域名，并CNAME至“mydomain.feedsky.com”（固定格式）。

何谓CNAME？
CNAME指别名记录也被称为规范名字,这种记录允许您将多个名字映射到同一台计算机。

在子域中添加子域，例：feed.vat58.com

废话不“多说”，wordpress之“广开言路”

发表于 2012-04-16 | 分类于 wordpress |

第三方社交评论服务如今已成为独立博客不可缺少的枢纽，将不同的社区，不同的人群联系起来，增强了博客与用户之间的黏性。评论更自由，交流更简洁。本文将展示“多说”评论系统，使wordpress“广开言路”。

在wordpress插件安装中，搜索“duoshuo”，找到多说社会化评论框插件安装启用，接下来按照你的喜好配置多说就可以了。

在多说中，可以同步wordpress原有评论，精彩不流失，友情不间断。从此可以轻松管理你的博客评论。

通过多说，你可以将多方帐号集结到你的麾下。你可以绑定多个多说帐号，人人网，QQ,各种微博，豆瓣网，MSN等等，并且可以在社交账号同步发布文章。

介绍了一个“多说”，泄漏了不少个人信息
“多说”果然让人 把不住嘴门啊。
正是由于这样的第三方社交工具，分享到，订阅到，同步到，登录到，互联网才如同一个巨大的蜘蛛网，千丝万缕联系你我她。
我倒没有多少个人隐私，乐意跟大家分享我的讯息。

类似的第三方社交评论系统还有友言，评论啦，等等。感兴趣的朋友可以自行比较，选取你中意的一款。
