初识正则表达式

发表于 2012-03-30 | 分类于 移动互联网 |

正则表达式（Regular Expression）: 是指一个用来描述或者匹配一系列符合某个句法规则的字符串的单个字符串。许多语言都支持正则表达式。它是一个超强的字符串处理的工具。

下面对正则表达式进行了详细的总结，便于查阅。

/ 将下一个字符标记为一个特殊字符、或一个原义字符、或一个向后引用、或一个八进制转义符

^ 匹配输入字符串的开始位置

$ 匹配输入字符串的结束位置

* 匹配前面的子表达式零次或多次

+ 匹配前面的子表达式一次或多次

? 匹配前面的子表达式零次或一次

{n} n 是一个非负整数,匹配确定的n次

{n,} n 是一个非负整数,至少匹配n次

{n,m} 最少匹配n次且最多匹配m次（n<=m,均为非负整数）

. 匹配除“\``n”之外的任何单个字符

x|y 匹配x或y

[xyz] 字符集合

[^xyz] 负值字符集合

[a-z] 字符范围

[^a-z] 负值字符范围

\b 匹配一个单词边界，也就是指单词和空格间的位置

\B 匹配非单词边界

\d 匹配一个数字字符，等价于[0-9]

\D 匹配一个非数字字符

\s 匹配任何空白字符，包括空格、制表符、换页符等。 等价于[ \f\n\r\t\v]

\S 等价于[^ \f\n\r\t\v]

当 ? 紧跟在任何一个其他限制符（ *, + , ? ，{n}，{n,}，{n,m}）后面时，匹配模式是非贪婪的。非贪婪模式尽可能少的匹配所搜索的字符串，而默认的贪婪模式则尽可能多的匹配所搜索的字符串。

简单示例：/^[a-z]{3}\d{2}$ / 匹配vat58

PHP之运算符

发表于 2012-03-27 | 分类于 PHP |

php运算符同其他语言的运算符大同小异。

算数运算符：+ - * / % ++ – 等。%取模运算符一般用来确定一个范围，比如对50求模，那结果肯定在50之内。

比较运算符：== === !=或<> !== < > <= >= ? : ===为全等运算符，比较的对象需要是值跟数据类型均相同才为真。对于!==不全等，只要值跟数据类型有一个不同就为真。

逻辑运算符：&& || & | ! 『 && ||』 跟 『& |』两组运算符都可以作逻辑运算，不同点在于前者存在短路，比如逻辑与&&，左边的逻辑值为假时，右边的表达式不再计算。

位运算符：& | ^ ~ << >> 首先将数据转换成二进制，然后按位进行相应的操作。

赋值运算符 = -= += /= %=

其他： . .= . 用来连接字符串。

WordPress之创建子主题

发表于 2012-03-25 | 分类于 wordpress |

Wordpress是一款强大的个人博客系统，如果你已经在写一个wp的博客，那么恭喜你：你己经站在了巨人的肩膀上。我们有理由把它完善地更好，更符合个人的要求和特色，所以有时候我们需要自己DIY。可你 有没有想过，你的“定制”和修改会随着下一次的升级付之东流，我确实担心过。

创建wordpress的子主题，会让这个问题迎刃而解。

WordPress子主题也是一个主题，它继承了另一个主题——父主题——的功能，并允许你对父主题的功能进行修改，或者添加新功能。

style.css是一个子主题唯一必须的文件。它的头部提供的信息让WordPress辨认出子主题，并且重写父主题中的style.css文件。

这样一个子主题就建好了。对一个父主题的样式和布局进行修改和扩展，而不需要对父主题的文件作任何修改。通过这样的方式，当父主题被更新的时候，所做的修改就可以保存下来。所以强烈建议以这种方式对主题进行修改。

你的子主题文件夹里面可以少至只包含一个style.css文件，也可以包含多至一个完整WordPress主题所拥有的文件：

1.style.css (必需)


/
Theme Name: Vinny
Theme URI: http://www.vat58.com/
Description: the test one by vinny_dong
Author: dong
Version: 1.0
Template: twentyten
/


以上头部信息解释为：


Theme Name. (必需) 子主题的名称 vinny。 Theme URI. (可选) 子主题的主页 www.vat58.com。
Description. (可选) 子主题的描述。比如：the test one by vinny_dong Author. (optional) 作者的名字 dong。
Template. (必需) 父主题的目录名，区别大小写。Version. (可选) 子主题的版本。比如：0.1，1.0，等。
可以用import命令将模板的css导入，然后修改样式规则，覆盖同名的样式规则

如：@import url(“../twentyten/style.css”); （此语句必须在所有规则之上才能起作用）

2.functions.php (可选)

不像style.css，子主题中的functions.php不会覆盖父主题中对应功能，而是将新的功能加入到父主题的functions.php中。（其实它会在父主题文件加载之前先载入。）
<?php

Function newFunction() {}

?>

可以以上述方式给主题添加功能。你也可以对父主题中的一个PHP函数重新声明替换。

3.Template files (可选)

在子主题中的表现和style.css一样，它们会覆盖父主题中的相同文件。子主题可以覆盖任何父主题模板中的文件，只需要创建同名文件就行。

相信接下来你肯定有很多工作要做了，定制一个*子主题，将以前改动的地方体现在你的子主题文件里，这样才可以后枕无忧。

php之数据类型

发表于 2012-03-23 | 分类于 PHP |

每门语言的前几章都是比较类似的基础知识，一个helloworld,讲讲数据类型，讲讲语法。什么时候才能抓住核心部分啊？不要急，step by step.这一步我们学习（回顾）php的数据类型。

在php概述一文中，我们提到，php是一种松散型的语言。也就是说，它比较“怂”。向来是“逆来顺受”，赋给它什么样的值，它都老老实实地接着，不知道反抗。

1.主要的数据类型有：

整型（int）：十进制，二进制，八进制，十六进制。怎么相互转换？忘了的自觉翻课本。

浮点数（float）:写到这里，我想到了计算机组成原理课本中，那些琐碎的题目。唉！零零碎碎，没完没了。这个也不多说。

字符串（string）:在php中，单引号 ‘’ 跟双引号 “” 都可以表示字符串。说到这里，你应该想到在其他语言中存在一个字符（char）类型,它就是用单引号 ‘’ 表示的。而在php中，字符型是不存在的。那单引号 ‘’ 跟双引号 “” 表示的字符串有什么区别吗？以后会提到。

布尔型（bool）:敞亮一点就是true跟false。另外，背地里还有不少：

整型非零为真，零为假；字符串中空””和”0”为假；其他为真;没有成员的数组为假;任何资源类型（什么是资源类型？）为真；空类型null代表假。大体上如此。

至于资源，对象等比较复杂的，以后细细品味。

2.两个取数据类型的函数：getType($a) 和var_dump($a)

3.强制数据类型转换：（php中也有自动转换的机制）

  • $b=（int）$a，形成新变量，元变量不变。$b是新变量，$a还是$a,不悲不喜。
  • setType($a,”int”) 原变量$a改变了，如果它之前不是int型。
  • $b=intval($a) 效果同第一种。类似方法还有floatval(); strval();
    最后提醒，转换时别忘了考虑溢出。大家可以动手试试强制转换一种到另一种，是什么规律。

晚安。

谈编程语言之“鸡毛蒜皮”

发表于 2012-03-20 | 分类于 PHP |

当你为了一个bug调试半天，缓过神来你才发现，竟是一个大小写的问题。也许你像我一样刚刚接触编程，亦或是对某种语言放下的时间太久，用到的时候总会出现一些鸡毛蒜皮的小事，却足以让人抓心挠肝。那么请接着往下看。

大小写敏感性：我接触的大部分语言都是大小写敏感的，像某些高级语言如Java，C++；php，jsp，javascript也都是大小写敏感的(php中只有变量对大小写敏感)。而像html,sql,vb则是比较迟钝的，呵呵。如果你不想理会这些，不妨都当作大小写敏感，规范编程，保险上上策。
命名规则：最基本的变量的命名规则：由字母，数字，下划线组成，且第一个元素不能为数字和下划线。具体到各种语言，细节就很多了。比如Java中包的命名，类的命名，函数的命名，都有相应的规范。基本的原则就是容易阅读，可维护性强。
