Android Fragment 专题

发表于 2014-09-09 | 分类于 Android |

Fragment(碎片)好比在Activity中嵌入的“小的Activity”, 每个小的Activity可以拥有自己的布局，拥有自己的生命周期，接受、处理事件，并且可以动态的添加、替换和移除。

XML中使用Fragment

下例Activity_main.xml作为主Activity的布局文件：

  1. <LinearLayout xmlns:android=”http://schemas.android.com/apk/res/android“
  2. android:layout_width=”match_parent”
  3. android:layout_height=”match_parent” >
    4.5. <fragment
  4. android:id=”@+id/fragment1”
  5. android:name=”com.vat58.demo.Fragment1” <!– 引用具体的Fragment类 –>
  6. android:layout_width=”0dip”
  7. android:layout_height=”match_parent”
  8. android:layout_weight=”1” />
    11.12. <fragment
  9. android:id=”@+id/fragment2”
  10. android:name=”com.vat58.demo.Fragment2” <!– 引用具体的Fragment类 –>
  11. android:layout_width=”0dip”
  12. android:layout_height=”match_parent”
  13. android:layout_weight=”1” />
    18.19. </LinearLayout>
    Fragment类及布局文件的代码不再展示了，类里加载布局文件，然后供主界面布局文件引用。

 

动态添加Fragment

程序运行时向Activity添加Fragment，程序的界面就可以定制的更加多样化。

因为要动态添加Fragment，所以要将Activity_main.xml中关于Fragment的引用删除，改为在代码里进行处理。下面例子的想法是，当手机竖屏时显示Fragment1，当手机横屏时显示Fragment2，达到动态添加的目的。

MainActivity示例：

  1. public class MainActivity extends Activity {
    2.3. @Override
  2. protected void onCreate(Bundle savedInstanceState) {
  3. super.onCreate(savedInstanceState);
  4. setContentView(R.layout.activity_main);
  5. Display display = getWindowManager().getDefaultDisplay();
  6. if (display.getHeight() > display.getWidth()) { //判断横屏的情况
  7. Fragment1 fragment1 = new Fragment1();
  8. getFragmentManager().beginTransaction().replace(R.id.main_layout, fragment1).commit();
  9. } else { //判断竖屏的情况
  10. Fragment2 fragment2 = new Fragment2();
  11. getFragmentManager().beginTransaction().replace(R.id.main_layout, fragment2).commit();
  12. }
  13. }
    16.17. }

Fragment的生命周期

 

[caption id=”” align=”aligncenter” width=”340”] Fragment生命周期对比Activity[/caption]

可以看到，Fragment与Acvitity的生命周期很像，重点解释下以下几个方法：

  • onAttach方法：Fragment和Activity建立关联的时候调用。
  • onCreateView方法：为Fragment加载布局时调用。
  • onActivityCreated方法：当Activity中的onCreate方法执行完后调用。
  • onDestroyView方法：Fragment中的布局被移除时调用。
  • onDetach方法：Fragment和Activity解除关联的时候调用。
