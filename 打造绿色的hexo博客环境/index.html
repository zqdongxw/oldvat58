打造绿色的hexo博客环境

发表于 2015-06-13   |   分类于 hexo   |  

##为何要搭建一个绿色的hexo博客环境

你可能会有N台机器，搭建一个绿色的环境，便与在不同机器上快速的开启HEXO写作之旅。

##博客根目录 ： X:/vat58

这个目录就是我们今后写作的大本营，绿色便携，包含了一切配置，只要同步它就可以了。

###下载Node.js

http://nodejs.org/download/

选择对应版本的Windows Binary(.exe)到X:/vat58

###下载npm源代码

https://github.com/npm/npm/releases/latest

选择Source code(zip)解压到X:/vat58/npm
执行 ：

X:\vat58/npm>node cli.js install -gf

完成编译。

###下载绿色版git安装hexo

https://github.com/msysgit/msysgit/releases/latest
选择PortableGit-1.9.5-preview20141217.7z解压到X:/vat58/git
运行git-bash.bat 安装hexo

npm install -g hexo

##同步X:/vat58

同步X:/vat58，将X:/vat58/npm 加入系统path路径（重启生效）。
用 hexo init 初始化一个目录或者用已有的Hexo目录。（如果在已有的hexo目录里进行初始化操作，会覆盖掉本地的_config.yml配置和其他配置）
这样就完成了本地hexo环境的复制。
到目前为止，绿色的hexo博客环境已经搭建完成。如果再遇到同步迁移的情况，进行本步操作即可。关于同步的方案，可以用dropbox或者点对点的同步工具。接下来配置ssh key，连接到github。

2015/6/16 记：

关于同步工具，折腾过微软的onedrive，几百兆的目录死活传不动，上行下载跟蜗牛一样。相比dropbox,我更看好点对点的同步工具 BT sync （version 1.4.111 , 不要升级到2.0）。

##重新配置SSH key 连接到github

gitbash里输入ssh-keygen -t rsa -C “邮件地址@youremail.com”

$ ssh-keygen -t rsa -C "邮件地址@youremail.com"
Generating public/private rsa key pair.
Enter file in which to save the key (/Users/your_user_directory/.ssh/id_rsa):<回车就好>

添加本设备对应的ssh key到github

打开本地C:\Documents and Settings\Administrator.ssh\id_rsa.pub文件,此文件的内容为刚才生成的密钥，准确复制。
在github设置里添加拷贝的ssh key。

现在你已经可以通过SSH链接到GitHub了，还有一些个人信息需要完善的。
Git会根据用户的名字和邮箱来记录提交。GitHub也是用这些信息来做权限的处理，输入下面的代码进行个人信息的设置，把名称和邮箱替换成你自己的，名字必须是你的真名，而不是GitHub的昵称。

$ git config --global user.name "xxx"//用户名
$ git config --global user.email  "xxx@gmail.com"//填写自己的邮箱

如果deploy的时候报错：Error: Deployer not found : github 则执行以下命令：

npm install hexo-deployer-git --save

一点点修葺

因为git是绿色的版本，每次都要启动git-bash.bat然后手动定位到博客目录（例如：F:\vat58\Hexo）, 然后执行相关的hexo指令。所以需要设置博客目录为git的默认HOME目录，减少每次重复的CD操作。修改git目录下etc/profile文件：

找到

# normalize HOME to unix path
HOME="$(cd "$HOME" ; pwd)"
export PATH="$HOME/bin:$PATH"

修改为

# normalize HOME to unix path
HOME="你想要修改的HOME路径"
HOME="$(cd "$HOME" ; pwd)"
cd
export PATH="$HOME/bin:$PATH"

再次启动git-bash.bat主目录就是博客目录了，别忘了把.ssh和.gitconfig文件从~拷贝到当前博客目录。这样每次只要启动git-bash.bat快捷方式就可以HEXO了。
