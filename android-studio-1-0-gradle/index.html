Android studio1.0升级后关于Gradle的问题及解决办法

发表于 2014-12-10   |   分类于 Android   |  

升级Android studio 1.0 后，碰到了如下的问题，分享下解决办法。

1. 由于被墙，无法下载gradle

android studio 卡在了_ “_gradle download http://services.gradle.org/distributions/gradle-1.12-all.zip” 这个进程上。

为什么会重新下载gradle呢？我发现在gradle的默认下载路径C:\Users\username.gradle\wrapper\dists\gradle-1.12-all里新增了一个文件夹。理论上只要把离线的gradle-1.12-all.zip放进去应该就OK了。我的具体操作是把旧文件夹里的gradle-1.12-all.zip放了进去。这个问题解决。但是接下来出现了问题2

 

  1. Gradle DSL method not found: ‘runProguard()’

升级android studio后，对gradle的依赖版本也进行了更新。这个问题的解决办法是，在app /src / build.gradle文件中，将runProguard()改为minifyEnabled()即可解决问题。
