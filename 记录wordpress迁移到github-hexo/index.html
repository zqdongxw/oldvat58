记录wordpress迁移到git+hexo

发表于 2015-05-21 | 分类于 wordpress hexo | |

第一次在编辑器下尝试用markdown语法写博客，边写边熟悉相关语法，同时专门记录一篇《markdown小白使用记录》。
本文记录wordpress博客vat58.com迁移至github和gitcafe的过程。

#为什么要迁移至git

github和gitcafe均提供了pages服务，用于介绍托管在上面的项目。由于空间免费，就有朋友想到了利用pages来搭建静态博客。我所见到的最早的对这方面的介绍应该是这篇《搭建一个免费的，无限流量的Blog—-github Pages和Jekyll入门》。本文采用的博客生成器不是Jekyll，而是Hexo。

wordpress是一款火爆的CMS系统，近年来越来越多的人感受到了它的臃肿。由wordpress迁移至git pages 的好处主要是：

  • 用markdown语法在自己喜欢的编辑器上更加轻便的写博客
  • 抛弃付费型空间且不限流量
  • 享受git的版本管理功能，不用担心文章遗失
  • 折腾万岁（有一定的技术门槛和耐心）

hexo是一个快速、简洁且高效的博客框架，优点如下：

  • node.js 所带来的超快生成速度，让几百个页面在几秒内瞬间完成渲染
  • 支持markdown
  • 一键部署到github或者gitcafe等网站
  • 丰富的插件

github由于GFW的存在，容易被墙或者访问速度收到影响，所以可以用国内的gitcafe代替或者备份。如果你要参照本文进行操作，请提前准备好github或gitcafe账号。

#本地环境搭建

依次安装

  • node
  • git
  • hexo

在gitbash里安装hexo:

npm install -g hexo   (npm:node包管理器)

#连接至github或gitcafe

##配置ssh key

gitbash里输入ssh-keygen -t rsa -C “邮件地址@youremail.com” 

$ ssh-keygen -t rsa -C "邮件地址@youremail.com"
Generating public/private rsa key pair.
Enter file in which to save the key (/Users/your_user_directory/.ssh/id_rsa):<回车就好>

注意1，此处的邮箱地址，你可以输入自己的邮箱地址；注意2，此处的「-C」的是大写的「C」
然后系统会要你输入密码，这个密码会在你提交项目时使用，如果为空的话提交项目时则不用输入。这个设置是防止别人往你的项目里提交内容。出现下图，说明配置成功。

##添加ssh key到github和gitcafe

  1. 打开本地C:\Documents and Settings\Administrator.ssh\id_rsa.pub文件,此文件的内容为刚才生成的密钥，准确复制。
  2. 在github和gitcafe对应的设置里添加拷贝的ssh key。

##创建用户信息

现在你已经可以通过SSH链接到GitHub了，还有一些个人信息需要完善的。
Git会根据用户的名字和邮箱来记录提交。GitHub也是用这些信息来做权限的处理，输入下面的代码进行个人信息的设置，把名称和邮箱替换成你自己的，名字必须是你的真名，而不是GitHub的昵称。

$ git config --global user.name "xxx"//用户名
$ git config --global user.email  "xxx@gmail.com"//填写自己的邮箱

#wordpress迁移及hexo相关配置

1.在wordpress后台导出全部内容的XML文件备用

2.部署Hexo

在我的电脑中建立一个名字叫「Hexo」的文件夹，然后在此文件夹中右键打开Git Bash。

$ hexo init

Hexo随后会自动在目标文件夹建立网站所需要的所有文件。

现在我们已经搭建起本地的hexo博客了，执行以下命令(在H:\hexo)，然后到浏览器输入localhost:4000看看。

$ hexo g
$ hexo s

附hexo常用命令：

hexo new "postName" #新建文章
hexo new page "pageName" #新建页面
hexo clean 
hexo generate #生成静态页面至public目录
hexo server #开启预览访问端口（默认端口4000，'ctrl + c'关闭server）
hexo deploy #将.deploy目录部署到Git

3.安装导入工具并将wordpress导入到hexo

命令如下：

$ npm install hexo-migrator-wordpress --save
$ hexo migrate wordpress source

source处为备用的wordpress xml 文件的路径。

4.为hexo更换主题

我选择的主题是NEXT

$ cd your-hexo-site
$ git clone https://github.com/iissnan/hexo-theme-next themes/next

接下来更改站点 _config.yml 中的 theme 字段设置为 next ，即可让主题生效。

主题更新：

cd themes/next
git pull

如果碰到git更新冲突的情况，学会使用stash：

git stash
#备份当前的工作区的内容，从最近的一次提交中读取相关内容，让工作区保证和上次提交的内容一致。同时，将当前的工作区内容保存到Git栈中。
git pull
git stash pop
#从Git栈中读取最近一次保存的内容，恢复工作区的相关内容。由于可能存在多个Stash的内容，所以用栈来管理，pop会从最近的一个stash中读取内容并恢复。

#详细配置站点的_config.yml文件

在next主题的github页面，有该主题目前支持的所有配置项，如下举例：

  • 使用多说

站点_config.yml,添加duoshuo_shortname字段，设置如下：

duoshuo_shortname: shortname

严重提示，此处的shortname非多说的用户名，请到多说后台管理设置里查找多说的二级域名。

  • 创建标签云

新建一个页面，命名为 tags 。命令如下：

hexo new page "tags"

编辑刚新建的页面，将页面的类型设置为 tags ，主题将自动为这个页面显示标签云。

  • 创建分类页面：

新建一个页面，命名为 categories 。命令如下：

hexo new page categories

编辑刚新建的页面，将页面的类型设置为 categories ，主题将自动为这个页面显示所有分类。

  • 添加 Google 百度 统计

编辑站点的 _config.yml ，新增字段 google_analytics 或者 baidu_analytics（取决于使用的统计系统）

google_analytics: your-analytics-id

baidu_analytics: your-analytics-id
  • 同时托管到gitcafe和github
deploy:
- type: github
  repository: git@github.com:zqdongxw/vat58.git
  branch: master

- type: gitcafe
  repository: git@gitcafe.com:zqdongxw/zqdongxw.git
  branch: gitcafe-pages

把上面的repository换成自己账户对应的即可。

用下面一条命令就可以将本地的博客部署到github和gitcafe上：

$ hexo d

#绑定独立域名进行域名解析

以gitcafe为例。

  • 在page项目管理页面，设置自定义域名
  • 到域名注册商管理页面，将DNS服务器更换为DNSpod的NameServers
  • DNSpod中完成域名解析
