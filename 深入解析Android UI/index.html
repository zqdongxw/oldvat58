深入解析Android UI

发表于 2014-08-20   |   分类于 Android   |  

View

所有的UI组件和一般2D图形的基础是VIEW类。这个通用类有很多的方法和属性来定义要显示的部件和图形实体，并且可以响应事件。一个Activity对应一个view对象，每一个view对象都有一个onDraw方法。onDraw可以把view对象绘在画布上。整个过程就是当需要更新UI时Activity会调用view对象的onDraw方法，一个按钮或者一个文本框，传给画布，然后渲染到屏幕上。

setContentView

如何把一个view对象展示在一个Activity窗口里？这就是setContentView方法。

Button button = new Button(this); setContentView(button);

这样充满整个屏幕的就是一个按钮。

Layout-ViewGroup

上个例子里边只有一个按钮，如果一个Activity只能显示一个view对象，那如何创建复杂的UI ? 那就要用到layou来存放其他的view对象。所以准确的说，一个activity对应的是一个layout view。

LinearLayout l = new LinerLayout(this); Button button = new Button(this); l.addView(button); setContentView(l);

这是通过代码的方式创建UI。
